# websearch 技能

## 概述
本技能提供结构化的网络信息检索能力。不仅执行搜索动作，更强调对用户意图的拆解、关键词的优化以及对搜索结果的真实性校验，最终输出带权威来源的简报。

## 核心规则
1. **禁止幻觉**：所有事实性陈述必须 100% 来源于搜索结果，绝不能使用模型自身的预训练知识来补充搜索结果中没有的具体数据（如时间、数字、版本号）。
2. **必须带源**：输出结论时，必须以 Markdown 链接形式标注来源 URL。
3. **勇于重试**：如果第一次搜索结果不相关、过时或数量不足，必须主动调整关键词重新搜索，最多允许迭代 3 次。

## 执行步骤

### 步骤 1：意图拆解与关键词构造
- 分析用户的自然语言问题，提取核心实体和约束条件。
- 将长句拆解为适合搜索引擎的短词组。
- *示例*：用户问“2024年新能源汽车销量排行”，构造关键词为：`"2024" "新能源汽车" "销量排行" 最新`。

### 步骤 2：执行搜索
在web_search.py中实现了搜索方式：
- `web_search(keywords: str, num_results: int = 1)`: 通用网络搜索，返回指定数量结果
- 可以使用"python web_search.py --keywords="人工智能最新发展" --num_results=2"来运行搜索
### 步骤 3：结果评估与二次检索（按需）
检查返回的搜索片段：
- **不通过**：如果结果全是广告、无关内容，或明显缺少最新信息 -> 调整关键词（加入“官方”、“教程”、“2024”等限定词），重新调用 `web_search`。
- **通过**：如果获得了至少 2-3 个高度相关的有效片段 -> 进入步骤 4。

### 步骤 4：综合输出
按照规定的输出格式组织最终回复。

## 输出格式
请严格按照以下 Markdown 格式输出：

### 🔍 搜索主题
[简短概括搜索的主题]

### 💡 核心结论
[用 1-3 句话给出最直接的答案]

### 📊 详细梳理
[分点详细说明，每一条事实后附加来源链接]
- 要点 1：具体内容描述... [来源]
- 要点 2：具体内容描述... [来源]

### ⚠️ 注意事项（可选）
[如果搜索结果存在局限、争议或时效性风险，在此提示用户]
